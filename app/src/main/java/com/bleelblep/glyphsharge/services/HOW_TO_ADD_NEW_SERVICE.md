# 🚀 Памятка: Как добавить новую функцию в Glyph Sharge

Процесс добавления новой фичи всегда состоит из **5 шагов**. Идем снизу вверх: от данных к интерфейсу.

## Шаг 1: Слой данных (`SettingsRepository.kt`)
Вся конфигурация фичи должна храниться здесь.
1. **Ключи и Дефолты:** Добавьте `KEY_FEATURE_NAME_...` и `DEFAULT_FEATURE_NAME_...` в `companion object`.
2. **Методы get/set:** Напишите методы для чтения и записи настроек (например, `isFeatureEnabled()`, `saveFeatureEnabled()`, `getDuration()`, и т.д.).
3. **Миграции и Инициализация:**
    * Добавьте дефолтное значение (выключено) в метод `applyFirstRunDefaults()`.
    * **Важно:** Увеличьте `KEY_LAST_MIGRATED_VERSION` и добавьте новый блок `if (lastMigrated < XXX)` в `applyVersionMigrations()`, чтобы у старых пользователей фича корректно проинициализировалась.
4. **Отладка:** Обязательно добавьте статус новой фичи в метод `dumpAllSettings()`.

## Шаг 2: Сервис и Логика (`services/MyNewFeatureService.kt`)
Если фича должна работать в фоне (слушать экран, батарею, уведомления):
1. Создайте класс наследующий `Service()`.
2. Добавьте аннотацию `@AndroidEntryPoint` (обязательно для Hilt DI).
3. Внедрите зависимости (`@Inject`): `SettingsRepository`, `GlyphAnimationManager`, и `GlyphFeatureCoordinator`.
4. **Управление жизненным циклом:**
    * Сделайте константы `ACTION_START` и `ACTION_STOP`.
    * В `onStartCommand` проверяйте включена ли фича и включен ли мастер-свитч глифов. Если нет — вызывайте `stopSelf()`.
5. **Координация Глифов (Важно!):** Перед запуском анимации *всегда* запрашивайте доступ через `featureCoordinator.acquire(GlyphFeature.YOUR_FEATURE)`. По завершении анимации вызывайте `release()`. Это предотвратит "перебивание" анимаций друг другом.

## Шаг 3: Связка с жизненным циклом (`MainActivity.kt`)
Чтобы фича запускалась вместе с приложением и реагировала на главный выключатель:
1. **Инициализация:** Напишите метод `initializeMyNewFeature()` (по аналогии с `initializePulseLock()`), который отправляет `Intent` на старт или стоп сервиса в зависимости от настроек.
2. **Регистрация:** Добавьте вызов этого метода внутри `initializeServices()`.
3. **Синхронизация:** Добавьте запуск/остановку сервиса в метод `syncServicesAfterToggle(glyphOn: Boolean)`. Если пользователь отключает Глифы целиком, ваш сервис тоже должен уснуть.
4. **Коллбэки для UI:** Создайте функции для интерфейса:
    * `testMyNewFeature()` (запуск демо-анимации через `lifecycleScope`).
    * `enableMyNewFeature()` (сохранить в настройки + запустить сервис + показать Toast).
    * `disableMyNewFeature()` (сохранить в настройки + остановить сервис + показать Toast).

## Шаг 4: UI Компоненты (`ui/components/NewFeatureCard.kt`)
Создайте карточку интерфейса по стандарту приложения (как `LowBatteryAlertCard` или `ScreenOffCard`):
1. **Визуальная часть:** Используйте `FeatureCard` внутри `Box` с анимацией нажатия (`scale`) и прозрачности (`alpha`).
2. **Индикатор состояния (resolvedTint):**
    * Фиолетовый (`0xFF674FA3`), если отключен главный сервис глифов.
    * Зеленый (`ACTIVE_GREEN`), если фича включена.
    * Красный (`INACTIVE_RED`), если фича выключена пользователем.
3. **Тумблер:** Добавьте поверх карточки `MorphingToggleButton` в правый верхний угол.
4. **Диалоговые окна:**
    * **Info Dialog:** Вызывается по клику на саму карточку. Содержит описание "Как это работает", кнопку "Тест" и кнопку "Настройки".
    * **Config Dialog:** Вызывается из Info-диалога. Содержит `Slider`, `FilterChip` (для анимаций), выбор звука.
    * *Хак для DI в Compose:* Если внутри диалога конфигурации нужно протестировать анимацию, доставайте `GlyphAnimationManager` через `EntryPointAccessors.fromApplication(...)`.

## Шаг 5: Интеграция в Главный Экран (`MainActivity.kt` -> `MainScreen`)
1. Прокиньте ваши методы (из Шага 3) в параметры `@Composable fun MainScreen`.
2. Найдите блок `LazyColumn` в маршруте `"home"`.
3. Вставьте вашу карточку внутри `FeatureGrid` (чтобы было по 2 в ряд) или отдельным `item { ... }` (если она занимает всю ширину).
4. Заверните передаваемые коллбэки включения/тестирования в функцию `requireGlyphService { ... }`. Это обеспечит появление Toast'а "Пожалуйста, включите сервис глифов", если пользователь пытается активировать фичу при выключенном мастер-свитче.

---

### 💡 Чек-лист проверки перед запуском:
- [ ] Переменная в `SettingsRepository` имеет дефолтное значение?
- [ ] Версия в `applyVersionMigrations()` обновлена?
- [ ] Сервис крашится, если нет прав на Foreground Service? (Не забудьте добавить манифест).
- [ ] Карточка становится тусклой и меняет цвет иконки, если выключить главный рубильник?
- [ ] Анимация использует `FeatureCoordinator`, чтобы не ломать другие анимации?
- [ ] Добавлены ли вызовы остановки `animJob?.cancel()` при закрытии приложения?

Следуя этому алгоритму, код останется чистым, а добавление новых фич будет занимать минимум времени!
